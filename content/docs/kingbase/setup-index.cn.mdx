## 安装方式

| 安装方式         | 描述                                                                                                                    |
| ---------------- | ----------------------------------------------------------------------------------------------------------------------- |
| 图形安装向导界面 | 用户通过 `./setup.sh -i swing` 的方式启动图形化安装向导                                                                 |
| 字符安装向导界面 | 用户通过 `./setup.sh -i console` 的方式启动字符化安装向导                                                               |
| 静默安装         | 用户通过 `./setup.sh -i silent -f install.properties` 的方式启动静默安装, install.properties 需放在 setup.sh 同级目录下 |

## 安装准备

### 按实施规范创建目录

1. 为了利于数据库的日常运维、持续使用、存储扩容等，我们在安装前须做好选项 存储目录规划。

| 选项        |                                                           设置                                                           |
| ----------- | :----------------------------------------------------------------------------------------------------------------------: |
| 目录        | 安装软件存储目录： `/install` 备份目录： `/backup` 归档目录： `/archive` 数据存储目录： `/data` KES 安装目录： `/kes/v9` |
| 端口        |                                                          54321                                                           |
| SYSTEM 密码 |                                                         kingbase                                                         |
| 数据库编码  |                                                           UTF8                                                           |

2. 实际操作

（1） 创建目录

```bash
mkdir -p /backup/dump /archive /data /KingbaseES/V9 /install
```

(2) 添加用户和组

```bash
useradd kingbase
```

（3） 设置目录的属主 属组 权限

```bash
chown -R kingbase:kingbase /install /backup /archive /data /KingbaseES
chmod -R 775 /KingbaseES /install /archive /backup
chmod -R 700 /data
[root@localhost ~]# ll  / | grep kingbase
drwxrwxr-x.   2 kingbase kingbase    6 4月   2 23:59 archive
drwxrwxr-x.   3 kingbase kingbase   18 4月   3 00:01 backup
drwx------.   2 kingbase kingbase    6 4月   2 23:59 data
drwxrwxr-x.   2 kingbase kingbase    6 4月   3 00:02 install
drwxrwxr-x.   3 kingbase kingbase   16 4月   3 00:12 KingbaseES
```

说明：

- **775**：权限为 `rwxrwxr-x`，目录属主和属组拥有读/写/执行权限，其他用户只有读/执行权限。这里用于安装目录、归档目录、备份目录等，方便同组运维人员协作管理。
- **700**：权限为 `rwx------`，只有目录属主有读/写/执行权限，其他用户无权限。这里用于数据库数据目录 `/data`，避免非 `kingbase` 用户访问数据文件。

(4) 上传安装盘 iso 检查 md5 挂载

```bash
md5sum /install/KingbaseES_V009R001C010B0004_Lin64_install.iso
mount -o loop /install/KingbaseES_V009R001C010B0004_Lin64_install.iso /mnt
```

(5) 使用 kingbase 用户复制挂载后的安装文件到 /install 目录

```bash
mkdir -p /install/KES-V009R001C010B0004-INSTALL
cp -r /mnt/* /install/KES-V009R001C010B0004-INSTALL/
```

修改所有者和组为 kingbase:

```bash
chown -R kingbase:kingbase /install/KES-V009R001C010B0004-INSTALL/
chmod -R 755 /install
```

(6) 上传授权文件

```bash
[root@localhost install]# ll
-rwxr-xr-x. 1 root     root           4998 4月   3 00:19 license_4_V009R001C-dev-365d.dat
mkdir -p /install/license
chown -R kingbase:kingbase /install/license
chmod -R 775 /install
```
